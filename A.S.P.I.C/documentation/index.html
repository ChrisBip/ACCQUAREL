Présentation générale de la plate forme A.S.P.I.C

Pourquoi faire encore un code de chimie ?

Avant de débuter un projet aussi titanesque que celui ci, il me semble nécessaire de répondre à cette question. Pour cela deux arguments s'imposent :

  • Si de nombreux codes existent déja, ceux ci s'adressent de façon quasi exclusive aux chimistes et peu aux informaticiens et aux mathématiciens suceptibles d'appliquer leurs connaissances à la chimie moléculaire.

  • De plus l'objectif principal de ces codes est généralement l'obtention d'un résultat numérique (énergie, force, ...) suceptible d'etre interprété par les chimistes, plus rarement de servir de tremplin à de nouveaux développements.

La plate forme A.S.P.I.C souhaite combler ce manque. Le but de cette "boite à outils" est de permettre aux chimistes et aux numériciens de développer et de tester de façon rapide et fiable de nouveaux algorithmes pour la chimie moléculaire.

Philosophie de développement.

Lors du développement de la plate forme, une attention toute particulière a été apportée aux trois point suivants :

Fiabilité.

Si le but est de tester de nouvelles méthodes, il faut que les résultats soient fiables : c'est à dire que l'echec d'un test ou une boucle d'optimisation qui ne converge pas doivent etre du à un problème conceptuel et non à une erreur de programmation. A.S.P.I.C a été élaboré pour conscrire au maximum les sources d'erreurs. 

Transparence.

La plate forme doit rester un outil et uniquement un outil. Il ne faut pas que le fait d'utiliser A.S.P.I.C pour un besoin entraine la nécessité de l'utiliser pour tout.

Nous souhaitons que le code soit accessible par le plus grand nombre, pour ce faire il faut aussi pouvoir étendre son champ de compétence de façon simple. Le principe consiste à laisser suffisamment de souplesse au code pour qu'il puisse évoluer.

Les solutions proposées.

Afin de répondre à ce "cahier des charges" la plate forme propose une solution 

Utilisation d'objets.

Les sources de la plate forme sont en C++ et les quantités manipulées sont des objets. Cela permet de manipuler les concepts comme des "boites noires" sans avoir à se préoccuper de la gestion des données interne à l'objet.

Modularité.

Le code se présente sous forme de modules qui peuvent etre réutilisé ou pas par l'utilisateur. Cela permet de ne pas imposer de séquence d'instructions et donc de laisser une grande souplesse à l'utilisateur final.

XML & Schémas

L'utilisation de fichiers XML permet de ne pas gérer, au sein de la plate forme, de format de donnée mais uniquement des informations. Suivant les utilisateurs intéressés une ou plusieurs interfaces pourront être rajoutées afin de formatter ces informations.

Utilisation d'outils informatiques appropriés au développement.

Dans la mesure du possible la plate forme utilise des programmes externe qui permettent, à moindre cout de développement, d'augmenter les fonctionnalités disponibles.

Pour l'instant nous utilisons

  • Doxygen et Dot pour la génération automatique de la documentation
  • Maple pour la validation des modules de calcul
  • CVS pour la gestion des versions.

Installation.

Prérequis.

Afin d'installer correctement le code fourni avec la plate forme A.S.P.I.C, deux bilbiothèques sont nécessaires :

  • lapack (et ces dépendances donc principalement blas et g2c). Cette biblitohèque est utilisé pour la résolution du système aux valeurs propres généralisé.

  • Xerces C++. Cette bibliothèque est utilisé pour la lecture des fichiers XML. Si cette bibliothèque n'est pas installée à un endroit standard, alors la variable d'environnement XERCESCROOT est utilisée pour spécifier le répertoire contenant cette librairie.

Dans ce qui suit, nous supposerons que ces biblitohèques sont installées sur la machine qui accueillera le code A.S.P.I.C.

Dépacketage.

Après avoir téléchargé une version du code A.S.P.I.C, placez vous dans le répertoire qui devra contenir A.S.P.I.C. Pour depacketer l'archive utilisez la commande :

% tar jxvf  <chemin vers le répertoire>/aspic-src.1.1.0.tar.bz2

Ensuite la varaible d'environnement $ASPICROOT est utilisée pour  que l'on puisse retrouvée de façon certaine le répertoire contenant les inforamtions relatives à la plate forme A.S.P.I.C. Afin de fixer cette valeur, du répertoire où les sources ont été depacketées :

% cd A.S.P.I.C

% export ASPICROOT=$PWD

Il est possible de vérifier que l'opération en effectuant l'opération suivante :

%echo $ASPICROOT

Compilation des librairies.

La compilation des librairies, s'effectue en trois étapes. Tout d'abord il faut se rendre dans le répertoire principal qui contient les sources de celles ci.

% cd $ASPICROOT/src

Ensuite il faut créer les librairies. 

% MODE=Release make (pour le mode de production).

% MODE=Debug make (pour le mode de debuggage).

Enfin, l'ensemble des entêtes peuvent etre regroupées (copiées) dans le répertoire $ASPICROOT/include à l'aide de la commande.

%make exportIncludes

Compilation des Exemples. 

Les exemples se trouvent dans le répertoire $(ASPICROOT)/sample & bin et se divise en plusieurs sous répertoire qui contiennent chacun un exemple d'utilisation d'une partie du code A.S.P.I.C.

Pour chacun de ceux ci, chaque répertoire se divise en deux parties : le répertoire src contient les  sources de l'exemple et le répertoire build contient le makefile qui permet de générer l'executable.

Une fois compilés, les binaires sont disponibles dans le repertoire $(ASPICROOT)/bin.

Génération de la documentation de classe.

Il est possible de générer de la documentation pour toutes les classes du code A.S.P.I.C. Cela se fait à l'aide de Doxygen.Un fichier de configuration pour cet utilitaire est disponible avec le code : $(ASPICROOT)/documentation/API/aspic.dox.

Les Bases de données

Généralités.

Une base de données est un ensemble structuré et organisé permettant le stockage de grandes quantités d'informations afin d'en faciliter l'exploitation (ajout, mise à jour, recherche de données). Une base de données se traduit physiquement par un ensemble de fichiers sur disque.

L'accès à une information contenue dans une base de données se fait par l'intermédiaire d'une interface à l'aide de la clé qui l'identifie de manière unique. pour chaque base de données, la plate forme fourni une interface  qui construit à partir d'un identifiant des objets C++ contenant les informations qui  se trouvent dans la base de données.

Pour la plate forme, la réalisation des bases de données a été faite à l'aide de fichiers XML. Cette solution a été choisie pour deux raisons : tout d'abord elle est simple à mettre en oeuvre (beaucoup plus simple qu'une base SQL), ensuite le XML peut etre facilement transcrit dans différents format à l'aide de XSLT ou de XML Query ce qui permet de présenter les données dans divers formats sans implémentations. 

La base de données des éléments chimiques.

Contenu

La base de données des éléments chimiques permet de stocker afin de pouvoir y accéder au sein d'un programme des données que l'on trouve généralement dans le tableau de Mendeleïev. Les données renseignées dans cette base sont disponibles sur cette page (fichier généré par le programme aspicDataBasePrinter).

Accéder à la base de données depuis un programme.

L'accès aux informations présnete dans cette base de donnée depuis un programme ce fait par l'intermédiaire de la classe chemicalElementsDataBaseInerface et la clé qui permet d'accéder à un élément chimique est généralement son symbole. Pour plus de précisions, le mieux est de se reporter à l'exemple qui illustre ce propos.

La possibilité de modifier cette base de donnée au sein d'un programme n'a pas implémentée.

Administrer la base de données.

L'ensemble des éléments chimiques est contenu dans le fichier XML :
  • $ASPICROOT/data/chemics/chemicalElements.xml
Le schéma qui régit la syntaxe de ce fichier est disponible dans le fichier :
  • $ASPICROOT/data/chemics/chemicalElements.xsd

La base de données des types de couches.

Contenu

La base de données des types de couches permet de stocker afin de pouvoir y accéder au sein d'un programme les données qui décrivent les différents types de couche en chimie moléculaire. Les données renseignées dans cette base sont disponibles sur cette page (fichier généré par le programme aspicDataBasePrinter).

Accéder à la base de données depuis un programme.

L'accès aux informations présnete dans cette base de donnée depuis un programme ce fait par l'intermédiaire de la classe xmlShellTypesDataBaseInerface et la clé qui permet d'accéder à un type de couche est le nom du type de couche. Pour plus de précisions, le mieux est de se reporter à l'exemple qui illustre ce propos.

La possibilité de modifier cette base de donnée au sein d'un programme n'a pas implémentée.

Administrer la base de données.

L'ensemble des types de couches est contenu dans le fichier XML :
  • $ASPICROOT/data/chemics/shellTypes.xml
Le schéma qui régit la syntaxe de ce fichier est disponible dans le fichier :
  • $ASPICROOT/data/chemics/shellTypes.xsd

La base de donnée des bases pour la chimie.

Contenu

La base de données des bases gaussiennes permet de stocker afin de pouvoir y accéder au sein d'un programme les données qui décrivent différents éléments chimiques dans une base de gaussiennes donnée. Les données renseignées sur dans cette base de données sont disponibles sur cette page (fichier généré par le programme aspicDataBasePrinter).

Il faut remarquer que certaines bases gaussiennes sont normalisées (celle dont la clé est en "nomDeBase-n") et d'autres ne le sont pas (la clé est en "nomDeBase"). Naturellement, les bases "nomDeBase" et "nomDeBase-n" contiennent les mêmes inforamtions (en terme de finesse de description des éléments chimiques). Les bases  normalisées ont été générées par le programme de nomralisation fourni avec la plate forme A.S.P.I.C.

Accéder à la base de données depuis un programme.

L'accès aux informations présentes dans cette base de donnée depuis un programme ce fait par l'intermédiaire de la classe xmlBasisDataBaseInerface et la clé qui permet d'accéder à la description d'un élément chimique dans une base est le  couple formé par le nom de la base et le symbole de l'élément chimique. Pour plus de précisions, le mieux est de se reporter à l'exemple qui illustre ce propos.

La possibilité de modifier cette base de donnée au sein d'un programme n'a pas implémentée.

Administrer la base de données.

Contrairement  aux bases de données pour les éléments chimiques et pour les types de couches, celle des bases gaussiennes ne se compose pas d'un seul fichier. En effet le choix a été fait de stocker chaque base (avec tout les éléments qu'elle contient) dans un document XML qui lui est propre. Ainsi, cette base de données comporte autant de fichiers qu'elle contient de bases chimiques.

Le répertoire qui contient tout les fichiers XML :
  • $ASPICROOT/data/chemics/gaussianBasis
Le schéma qui régit la syntaxe de chaque fichier de base est disponible dans le fichier :
  • $ASPICROOT/data/chemics/gaussianBasis.xsd

Les Bibliothèques.

Généralités

La plate forme étant destinée particulièrement aux personnes qui souhaitent tester de nouvelles méthodes en chimie moléculaire, il a été choisi de proposer principalement des bibliothèques sur lequelles les développeurs pourront s'appuyer.

Le propos est ici de présenter succinctement les fonctionnalités proposées par chacune d'elles. Les personnes souahitant une documentation détaillée de chaque classe peuvent se reporter directement à l'API.

La bibliothèque  aspicUtils.

Le rôle de cette bibliothèque est de fournir une aide substantielle pou la manipulation des données au sein de programmes qui s'appuient sur la plate forme. En fait cette bibliotèque contient  la classe aspicConfiguration et la classe cmdLineParser.

La classe cmdLineParser est un utilitaire qui facilite la gestion des arguments passés en ligne de commande.

La classeaspicConfiguration  permet de retrouver, au sein d'un programme les différents chemins vers les fichiers de données qui viennent avec la plate forme.

La bibliothèque chemics.

La bibliothèque chemics implémente toutes (ou presque) les classes nécessaires à la manipulation d'objets pour la chimiqe moléculaire ainsi que les interfaces qui permettent de lire les informations correspondantes à ces objets dans des fichiers XML.

La bibliothèque complex.

gestion des nombre complexes.

La bibliothèque containor.

gestion de tout ce qui est tableau dynaique ou statiques.

La bibliothèque gaussian.

bibliothèque de modélisation des fonction gaussienne mono et tri dimentsionnelles, ainsi que des produits de polynomes et de gaussiennes.

La bibliothèque gintegrals.

bibliothèques pour calculer les intégrales de la chimie.

La bibliothèque model.

gestion des modèles chimiques. A étoffer...

La bibliothèque matrix.

gestion simplissime des matrices. Ici elle est utilisé pour des raisons de debugages.

La bibliothèque polynome.

gestion des polynomes.

La bibliothèque xmlParser.

couplage du code aspic avec un parser XML (dans notre cas nous avons choisi XERCES C++).

Binaires et Exemples.

aspicDataBasePrinter.

Principe.

Le but de se programme est de créer des fichiers html qui synthétisent les informations présentes dans les bases de données fournies avec la plate forme.

Entrées.

Ce programe ne nécessite pas de fichier d'entrée.

Sorties.

Ce programme génère les trois fichiers :

  • $(ASPICROOT)/documentation/chemicalElements.html
  • $(ASPICROOT)/documentation/shellTypes.html
  • $(ASPICROOT)/documentation/gaussianBasis.html

gaussianBasisNormaliser.

Principe.

Souvent les données fournies pour les bases gaussiennes ne corresponde pas à des coefficients de base normaisées (telles que le produit scalaire de deux fonctions de bases soit égal à 1). Mais,  pour des raisons numériques, il est parfois nécessaire de travailler avec des fonctions de base normalisées. A.S.P.I.C propose un programe qui, à partir d'une base gaussienne quelquonque, crée une base normalisée.

Entrées.

Un fichier XML qui contient les données pour une base gaussienne quelquonque.

Sorties.

Un fichier XML qui contient la même base que le fichier d'entré mais dont les coefficients de contractions ont été normalisés.

getChemicalMatrixes.

Principe.

Ce programme calcule les différentes composants matriciels du hamiltonien d'une molécule.

Entrées.

Les entrées de ce programme sont :
  • Un fichier XML qui contient la description d'un système moléculaire.
  • Le format dans lequel les données de sortie doivent être écrites. Il existe à pour le moment trois formats disponibles. Tout d'abord le format huamain génère un fichier facilement lisible par l'homme.Enuite le format texte est un format pour lequel les information peuvent etre relues facilement dans un programme externe à condition de connaitre l'ordre dans lequel les données ont été écrites. Enfin le format XML génère un fichier au format XML dont les données peuvent etre réutilisées au sein de la pkate forme.
  • La liste des matrices à générer. Cette liste peut comporter cinq éléments : le recouvrement; l'énergie cinétique, l'énergie potentielle, le hamiltoinien et le tenseur de coulomb.
  • Le nom du fichier dans lequel les matrices calculées devront etre écrites.

Sorties.

Le programme génère un unique fichier texte. Le format de ce fichier est celui qui a été demandé par l'utilisateur (humain , texte ou XML). Les données contenues dans ce fichier sont une description la molécule pour laquelle les matrices ont été générées, ainsi que toutes les matrices demandées par l'utilisateur (recouvrement , énergie cinétique, énergie potentielle, hamiltonien, tenseur de coulomb).
